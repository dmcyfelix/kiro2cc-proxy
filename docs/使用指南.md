**前置要求：**

| 工具 | 安装命令 |
|------|----------|
| Node.js / npm | `brew install node` |
| Rust / cargo | `curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs \| sh` |

构建完成后，二进制位于 `./target/release/kiro2cc-proxy`，后续无需重新构建（除非更新了代码）。

## 启动与停止

### 启动

```bash
./run-local-service-mac.sh
```

**首次启动**（`app/config/config.json` 不存在时）会进入配置向导：

```
  API Key（访问此代理的密钥，自定义即可，必填）:
  Admin API Key（管理后台密码，直接回车跳过，可选）:
  端口 [默认: 5678]:
  Region [默认: us-east-1]:
  本地 HTTP 代理端口（直接回车跳过）:
```

配置完成后自动生成 `app/config/config.json`，服务启动。若配置了 Admin API Key，2 秒后浏览器自动打开管理面板；未配置则 Admin API 整体不启用。

**后续启动**直接读取已有 `app/config/config.json`，无需重新配置。

### 停止

在运行服务的终端窗口按 `Ctrl+C`。

## 配置说明

配置文件实际路径为 `app/config/config.json`（`run-local-service-mac.sh` 通过 `--config` 参数指定；参考仓库根目录的 `config.example.json`）：

```json
{
  "host": "127.0.0.1",
  "port": 5678,
  "apiKey": "sk-kiro2cc-proxy-qazWSXedcRFV123456",
  "tlsBackend": "rustls",
  "region": "us-east-1",
  "adminApiKey": "sk-admin-your-secret-key"
}
```

| 字段 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `apiKey` | **是**（缺失时进程直接退出） | — | 号池 apiKey，客户端连接时使用，不要泄露给他人 |
| `adminApiKey` | 否 | — | Admin API 密钥（管理面板登录用）；未配置时 Admin API 整体不启用 |
| `host` | 否 | `127.0.0.1` | 监听地址，改为 `0.0.0.0` 允许局域网访问 |
| `port` | 否 | `8080`（本地启动脚本默认写 `5678`） | 监听端口 |
| `region` | 否 | `us-east-1` | AWS 区域 |
| `authRegion` | 否 | 同 `region` | Token 刷新区域 |
| `apiRegion` | 否 | 同 `region` | API 请求区域 |
| `proxyUrl` | 否 | — | HTTP/SOCKS5 代理，如 `http://127.0.0.1:7890` |
| `proxyUsername` / `proxyPassword` | 否 | — | 代理认证用户名/密码 |
| `loadBalancingMode` | 否 | `priority` | `priority`（按优先级）或 `balanced`（轮询） |
| `tlsBackend` | 否 | `rustls` | `rustls` 或 `native-tls` |
| `maxRpmPerCredential` | 否 | `0`（不限制） | 单账号每分钟最大请求数，超出排队等待 |
| `machineId` | 否 | 自动生成 | 设备标识 |
| `kiroVersion` / `systemVersion` / `nodeVersion` | 否 | 内置默认值 | 上报给 Kiro 上游的客户端版本信息 |
| `countTokensApiUrl` / `countTokensApiKey` / `countTokensAuthType` | 否 | — | 外部 `count_tokens` API 配置（`countTokensAuthType` 默认 `x-api-key`） |
| `cacheSimulation` | 否 | 内置默认值 | Prompt cache 模拟与指纹追踪相关配置（详见 `docs/Kiro缓存与状态管理解析.md`） |

修改 `app/config/config.json` 后重启服务生效。若需重新配置，删除该文件后重新运行 `./run-local-service-mac.sh`。

## 凭据配置

凭据文件实际路径为 `app/config/credentials.json`（同样由 `run-local-service-mac.sh` 通过 `--credentials` 参数指定），支持单账号和多账号两种格式。

### Social 登录账号（单账号）

```json
{
  "refreshToken": "your-refresh-token",
  "expiresAt": "2025-12-31T02:32:45.144Z",
  "authMethod": "social",
  "machineId": "64位十六进制字符串"
}
```

### IDC 账号（单账号）

```json
{
  "refreshToken": "your-refresh-token",
  "expiresAt": "2025-12-31T02:32:45.144Z",
  "authMethod": "idc",
  "clientId": "your-client-id",
  "clientSecret": "your-client-secret",
  "region": "us-east-2",
  "machineId": "64位十六进制字符串"
}
```

### 多账号（数组格式）

```json
[
  {
    "refreshToken": "token-1",
    "expiresAt": "2025-12-31T02:32:45.144Z",
    "authMethod": "social",
    "machineId": "aaa...aaa",
    "priority": 0
  },
  {
    "refreshToken": "token-2",
    "expiresAt": "2025-12-31T02:32:45.144Z",
    "authMethod": "idc",
    "clientId": "xxx",
    "clientSecret": "xxx",
    "region": "us-east-2",
    "machineId": "bbb...bbb",
    "priority": 1,
    "disabled": false
  }
]
```

`priority` 数值越小优先级越高；`disabled: true` 暂时禁用该账号；`machineId` 可省略，未提供时自动生成。仓库未附带 `credentials.example.*.json` 示例文件，请直接参考上方三种格式手动创建 `app/config/credentials.json`。

## 管理面板

服务启动后访问：`http://127.0.0.1:5678/admin`

访问时需要输入 `app/config/config.json` 中配置的 `adminApiKey`。若未配置该字段，Admin API 整体不启用，管理面板无法访问。

管理面板功能：
- 查看已加载的凭据列表及状态
- 启用 / 禁用单个凭据
- 调整凭据优先级
- 查看各账号余额
- 重置账号状态

## Claude Code 接入

服务启动脚本不会自动配置 Claude Code，需要手动设置环境变量或写入 `settings.json`。

### 方式一：环境变量

```bash
export ANTHROPIC_BASE_URL="http://127.0.0.1:5678"
export ANTHROPIC_API_KEY="你配置的 apiKey"
```

加入 `~/.zshrc` 可永久生效：

```bash
echo 'export ANTHROPIC_BASE_URL="http://127.0.0.1:5678"' >> ~/.zshrc
echo 'export ANTHROPIC_API_KEY="你配置的 apiKey"' >> ~/.zshrc
source ~/.zshrc
```

### 方式二：settings.json 配置

在 `~/.claude/settings.json`（全局）或 `<项目根目录>/.claude/settings.json`（项目级）中添加：

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "http://127.0.0.1:5678",
    "ANTHROPIC_API_KEY": "你配置的 apiKey"
  }
}
```

详见 `README.md` 的「接入 Claude Code」章节。

## 常见问题

**Q: 启动报错 `配置文件中未设置 apiKey`**

删除 `app/config/config.json` 重新运行 `./run-local-service-mac.sh`，按向导重新配置。

**Q: 已加载 0 个凭据配置**

需要创建 `app/config/credentials.json`，参考上方凭据配置章节。

**Q: Claude Code 请求返回 401 Unauthorized**

API Key 不匹配。检查是否已按上方「Claude Code 接入」章节设置了 `ANTHROPIC_API_KEY` 环境变量或 `settings.json`，且与 `config.json` 中的 `apiKey` 一致。

**Q: 端口被占用**

`run-local-service-mac.sh` 会自动杀掉占用配置端口的进程。如仍报错，手动执行：
```bash
lsof -ti:5678 | xargs kill -9
```

**Q: 局域网内其他设备无法访问**

`host` 默认是 `127.0.0.1`（仅本机可访问），需在 `app/config/config.json` 中手动改为 `0.0.0.0` 并确认防火墙未拦截对应端口。

**Q: 构建时 npm install 卡住**

脚本已配置使用淘宝镜像 `registry.npmmirror.com`。如仍卡住，检查网络连接后重试。

**Q: 如何更新代码后重新构建**

```bash
git pull
./build-mac.sh
./run-local-service-mac.sh
```
